# Maturity Levels: Paperhat

- **Status**: NORMATIVE
- **Lock State**: UNLOCKED
- **Version**: 1.0.0

## Purpose

Defines the general-purpose Paperhat specification maturity track. Suitable for any organization that does not follow a specific standards body process.

## Maturity Stages

| Token | Label | Description |
|---|---|---|
| `$Draft` | draft | Initial working document, subject to change without notice. |
| `$Review` | review | Feature-complete draft undergoing formal review. |
| `$Stable` | stable | Approved, normative specification ready for implementation. |
| `$Deprecated` | deprecated | Superseded or obsoleted; retained for historical reference only. |
| `$Retired` | retired | Permanently withdrawn from active use. |

## Design Decisions

- Five stages cover the full lifecycle from inception through retirement.
- `Review` is distinct from `Draft` to signal that content is feature-complete and soliciting feedback.
- `Deprecated` and `Retired` are separate: deprecated specs may still be referenced; retired specs should not.
